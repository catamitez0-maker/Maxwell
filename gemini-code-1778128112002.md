# ⚡️ Maxwell Protocol: Project Development Plan

> "Stop paying for Token length. Pay for real FLOPs."
> 停止为信息载体买单，为真实的物理做功付费。

## 1. 项目愿景 (Vision)
打造一套去中心化的开源算力结算与过滤标准。通过“启发式剪枝”技术拦截无效探测请求，并通过底层算法精确计算 AI 推理的真实浮点运算量（FLOPs），为构建绝对公平的 AI 算力网络提供底层基础设施。

## 2. 核心架构设计 (Core Architecture)
本项目采用高度解耦的模块化设计，分为三大核心子系统：

### 2.1 `maxwell-proxy` (启发式算力剪枝网关)
部署在主算力引擎之前，基于纯异步 I/O 的超轻量级防线。
* **L1 过滤层:** 基于内存的 Bloom Filter，以 $O(1)$ 的时间复杂度极速拦截历史失败特征或已知无效哈希。
* **L2 过滤层:** 正则匹配与长度/格式等数学特征校验。
* **L3 过滤层:** 基于 Shannon Entropy (香农信息熵) 计算，剥离无意义的循环字符与绝对乱码。

### 2.2 `maxwell-oracle` (算力测量预言机)
AI 推理算力的“精准标尺”与防作弊探针。
* **前置预估:** 根据大模型参数规模 ($N$) 和输入上下文，毫秒级计算基础 FLOPs 消耗，并设定该任务的 `FLOPs_Limit`（算力熔断天花板）。
* **防欺诈校验:** 分析底层特征或验证计算快照，识别并拦截死循环空转脚本，确保系统执行的是“有用工作量 (PoUW)”。

### 2.3 `maxwell-contracts` (结算共识模板)
面向去中心化算力网络的智能合约层。提供标准化的按 FLOPs 扣费逻辑，为算力定价提供无需信任的账本支持。

## 3. 技术栈与核心依赖 (Tech Stack & Dependencies)
系统底层逻辑强调高并发、极简内存占用与强类型安全。

### 3.1 核心运行环境
* **语言:** Python 3.10+ (强制开启并遵守 Type Hints 规范)
* **并发模型:** `asyncio` (全量异步非阻塞 I/O 处理，严禁使用传统阻塞型线程锁)

### 3.2 依赖包 (Dependencies)
* **CLI 与交互层:**
  * `typer` 或 `argparse`: 构建健壮的命令行解析入口。
  * `rich`: 构建终端实时监控仪表盘（显示总请求、拦截率、当前信息熵阈值）。
* **核心算法与数据结构:**
  * `bitarray` + `mmh3` (MurmurHash3): 用于徒手构建极致内存性能的 Bloom Filter。
  * `numpy` / `math`: 用于实现高精度的对数运算与香农信息熵概率分布计算。
* **开发与测试测试组件:**
  * `pytest` + `pytest-asyncio`: 用于异步并发队列的单元测试与压力测试。
  * `mypy`: 静态类型推导防范潜在 Bug。

## 4. 分阶段开发路线图 (Development Phases)

### Phase 1: MVP 代理层骨架与核心过滤漏斗
* 搭建基础 CLI 命令层与 `asyncio` 的异步生成器/消费者数据流通道。
* 编写 L1 模块：完成 Bloom Filter 类的构建、哈希映射与状态闭环测试。
* 编写 L3 模块：实现字符串信息熵计算函数，支持动态传入阈值进行拦截测试。

### Phase 2: 算力测算引擎 (Oracle 雏形)
* 梳理并用代码实现基于 Transformer 架构的静态 FLOPs 预估公式。
* 实现输入数据体积到算力天花板（`FLOPs_Limit`）的动态换算模块。

### Phase 3: 动态水位压制与探针策略
* 引入系统自适应机制：感知内部处理队列压力，高负载时自动调高信息熵剪枝阈值（严格模式）。
* 设计防空转逻辑的特征提取模型。

### Phase 4: 结算协议与开源打包
* 采用 Solidity 编写标准的结算合约模板。
* 完善项目文档结构，封装可用模块并发布。
